## Executive summary

This PR is a **near-complete, high-quality adoption of the flywindy/o11y SDK** across the whole platform. All 14 services (plus the 2 data-migration tools) are converted to `pkg/obs.Init`; the bespoke `pkg/otelutil` and the direct `Marz32onE/otel-nats` dependency are gone from first-party code; Mongo/Cassandra/Valkey/Elasticsearch/Vault/NATS/Gin/Resty are all instrumented through consumer-defined option seams; trace continuity across NATS (publish→consume, request→reply, JetStream fetch) is covered by dedicated integration tests. **Estimated o11y integration completeness: ~90%** (per-area breakdown in the Observability chapter).

The dominant risks are operational, not correctness: telemetry-flush ordering is inconsistent in 3 services (worst: search-sync-worker flushes **before** NATS drain), every service now **crashes at startup if `OTEL_SERVICE_NAME` is unset** (a deploy prerequisite living outside this repo), the repo's local compose stack **defines no otel-collector/Tempo/Grafana** despite defaulting every service to `http://otel-collector:4318`, and early-startup fatals regressed from JSON to Go's default text logger in every service that dropped its `slog.SetDefault` bootstrap line.

Reviewability: the code itself is disciplined and consistent, and the 27 commits are well-scoped — but the PR description still says "**planning/docs gate, no production code changes land here**" while the branch ships the entire Phase 0–4 implementation in one PR, abandoning its own locked per-phase rollout plan. The description must be rewritten (or the PR split) before review sign-off; as it stands a reviewer approving "docs only" would be approving 4,900 lines of production change.
